###开发指南
创建该项目的目的是开发一个app，可以和有公网ip的服务器web端通信，从而控制嵌入式设备。
功能需求：
【1】登陆界面：
考虑使用对称密钥加密，这样用户只需要在登陆界面输入经过加密后，包含ip、port、用户名和密码的信息即可登陆控制指定的设备，
用户名在这里相当于是设备id了，保证自己的设备只能自己控制，不会出现乱套。只有root用户才能修改其他用户的密码，其他用户没有修改密码的权限，
修改自己账户的密码也不被允许。
首次登陆才需要输入加密链接，
登陆成功后下次打开app就可以直接使用了，将加密链接保存在手机本地即可。
如果登陆失败，需要提示登陆失败，提醒用户切换其他有效加密链接
【2】root账户界面：
只有当前登陆的用户是root用户才展示该界面，该界面的功能有：添加新用户和密码的同时生成登陆所需的加密链接、删除用户、修改用户密码\
查询指定用户的密码和加密链接、查询指定用户的登陆以及近五次操作记录、查询所有用户最近一次的登陆时间
【3】切换模式界面
登陆以后就展示该界面，有4个模式：时间、日期、发送文本和加减乘除，默认激活时间模式；其中加减乘除和发送文本模式完成操作后会停留在最后显示的地方，这两个步骤也可以中途停止，
但是硬件设备已经在执行的指令就一定要执行到位了，还没开始执行的话就可以不执行。
点击切换模式的时候，操作者需要付出一些时间代价，以免频繁切换消耗过多服务器资源，同时使硬件设备的步进电机过热，建议，可以在切换过程中添加一个验证步骤，
消耗使用者一点时间，从而达到间隔切换模式的目的。
	【3-1】发送文本界面	
	因为客户端app处在后台以后，执行到一半的就有可能被干掉进程了，所以需要一次性将想要显示的文本都发到服务端，
然后服务端再自己制定发送任务，同时还需要及时将发送的状态回传给客户端。
	最多可以一次性连续发送3段5字文本，显示在屏幕的最上层，标好发送的顺序，可以删除不想要发送的文本，页面的大致样式如下：
1 HELLO ➖
	...
  添加 or 停止		【编辑文本过程中：当还没有输入满3段文本的时候，添加按钮都会显示；执行过程中：显示停止按钮】
文本编辑区
	【3-2】加减乘除界面
	由客户端生成随机的算式，然后一次性发送给服务端，服务端再自己制定发送任务，同时还需要及时将发送的状态回传给客户端。
	页面的大致样式如下：
XX + XX = XXXX 执行完毕
XX + XX = XXXX 待执行
停止按钮

开始游戏和结束游戏以后，都会在正中间显示一个启动按钮
【4】验证界面
防止用户频繁发送指令，需要像获取手机验证码一样有一个客户端验证的过程，这个验证是有等级的，分为难易两个等级，难：消耗时间更多；易：消耗时间少。
就使用加减乘除吧，如果是难，可以多个算式，每个算式间隔一点时间,还需要考虑一点：验证时间内设备已经执行完操作，
这时候就不需要再向设备发出停止操作的指令了。
